Custom Animation

This workflow includes creating custom animation in a DCC application and Importing the animation into the unreal engine as a .fbx file. This can be done with any DCC application that allows the user to export a .fbx file. i.e. Maya, Blender 3dsmax etc.

Keywords: Custom Rig, DCC application, FBX


Pain Points

  • Proficient animation skills are required for high-quality results in this workflow.
  • Creating hyper-realistic animations can be challenging.
  • The Unreal Engine may produce errors when trying to access the keys after importing.
  • Occasionally, imported textures may be deformed or not visible. Correctly UV unwrapping the assets can solve this issue.
  • Simulations or particle systems that are part of the character animation may not be visible when imported as .fbx. Importing them as Alembic files can resolve this issue.
  • Complex rigs with smooth corrective deformers or warping may not work when imported as .fbx. The .fbx file format supports bones, bone weights, and blend shapes.


Best Practics

  • Ensure custom rigs only contain bones, weights, and blend shapes, without smooth corrective deformer or warping.
  • Refrain from altering the rig during the animation process.
  • For simulations and particles, use the Alembic export method.
  • When importing, adjust the Up Axis to follow the Z-axis to allow for a compatible import into Unreal Engine.
  • Maintain organization in the scene or hierarchy with consistent naming conventions for all objects, bones, and animation in the DCC application.
